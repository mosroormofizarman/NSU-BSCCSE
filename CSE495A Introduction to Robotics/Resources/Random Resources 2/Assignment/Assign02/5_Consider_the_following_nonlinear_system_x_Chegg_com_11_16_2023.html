6. The A* grid-based search algorithm's steps are as follows:

1. Create an open list and a closed list, two sets of nodes that hold the potential paths and the paths that have already been examined, respectively.

2. Include the initial node in the open list and give it the value f, which is the product of the initial node's value and the values g and h. going from the starting node to the current node costs g, and going from the current node to the destination node costs h, as calculated by a heuristic function.

3. Continue until the goal node is located or the open list is empty:
- Locate the node in the open list with the lowest f value and designate it as q.
- Add q to the closed list and take it out of the open list.
- Create q's successors, which are the nearby nodes that are unobstructed, and assign q as their parent.
- For every succeeding:
- If it's the destination node, halt your search and go backwards to determine the best route.
Calculate its g, h, and f values if not.
- Skip it if it has already appeared in the open list with a lower f value.
- Skip it if it has already been added to the closed list with a lower f value.
- If not, include it on the open list.


Explanation:
The A* algorithm is a heuristic function-based informed search method that employs to direct its exploration. If the heuristic function is acceptable (never overestimates the cost) and consistent (satisfies the triangle inequality), the best path will always be discovered. It does, however, have several shortcomings, such as:

- Because it keeps all the created nodes in memory, it could be memory-intensive.
- If the search space is big or complex, or if the heuristic function is not well-chosen, the process may be sluggish.
- It might not be complete if the graph has cycles or infinitely many nodes with the same f value.

7. The stages for the sampling-based Probabilistic Road Map (PRM) mobility planning method are as follows:

1. Create a series of random samples in the robot's configuration space, eliminating those that collide with barriers.
2. Use a local planner, such as a straight-line planner, to connect each sample to its closest neighbours. Then, add the successful connections to a roadmap graph.
3. Keep going back and forth between steps 1 and 2 until the roadmap graph is thick enough to show the connectedness of the open area.
4. Using a graph search technique like A*, locate a path from the start to the goal on the roadmap graph given a start and a goal configuration.
5. You may choose smooth out the way to raise its standard.

The PRM technique is a sampling-based motion planning system that can deal with complicated barriers and high-dimensional configuration spaces. It works particularly well in circumstances involving many queries since the same roadmap may be applied to various start and goal setups. It does, however, have several shortcomings, such as:

- It can be ineffective in crowded or confined spaces where many samples are lost or don't connect.
- The selection of parameters, such as the quantity of samples, the connection radius, and the local planner, may have an impact on it.
- It may result in pathways that are not ideal or natural and require further smoothing or improvement.

8. The steps for using the sampling-based Rapidly Exploring Random Tree (RRT) motion planning approach are as follows:

1. Create a tree from scratch with just one node when the robot is configured at the beginning.
2. Continue doing so until a desired configuration is attained or a set number of iterations has been reached:
- Draw a random configuration at random from the robot's state space.
- Employ a distance measure to locate the node in the tree that is closest to the sampled configuration.
- Using a local planner, such as a straight-line planner, extend the tree from the nearest node towards the sampled configuration, and if the new node and edge are collision-free, add them to the tree.
- Determine if the new node is in a goal region or sufficiently near to a goal configuration, and stop the search if it is.

The RRT technique is a sampling-based motion planning algorithm that is capable of navigating complicated barriers and high-dimensional state spaces. In single-query circumstances when the start and target parameters are established and known in advance, it is particularly appropriate. It does, however, have several shortcomings, such as:

- It can be ineffective in crowded or confined spaces where many samples are lost or don't expand.
- It may depend on the settings you choose, such as the local planner, the distance metric, and the sample distribution.
- It may result in pathways that are not ideal or natural and require further smoothing or improvement.
